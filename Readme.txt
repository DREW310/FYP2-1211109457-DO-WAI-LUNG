# Detection of Foodborne Pathogens using Deep Learning

Source code package for the FYP2 project:

**Binary cocci-vs-bacilli morphology classification with GAN minority balancing.**

The main task is binary classification:

- `cocci = 0`
- `bacilli = 1`

The project uses source-level splitting to prevent data leakage. Source images are assigned to folds before patches are extracted, so patches from the same source image are not used in both training and testing in the same round.

## Repository contents

This repository contains the source code and small audit files needed to understand and rerun the project:

- `notebooks/` — Jupyter notebooks for data checking, splitting, GAN training, CNN training, comparison, and final figure generation.
- `manifests/` — final slide, fold, patch, and GAN-generated patch manifests.
- `figures/` — final report figures generated from the experiment results.
- `results_summaries/` — small metric summaries and comparison outputs. Large prediction files and training histories are not included.
- `third_party/` — local copies of the GAN source-code dependencies used by the notebooks.
- `requirements.txt` — main Python package list.

Large generated files are not stored in GitHub:

- raw datasets
- extracted 224×224 patches
- GAN training patches
- generated GAN images
- CNN checkpoints
- GAN checkpoints
- full prediction files
- full training histories

These files are excluded because they are large and can be regenerated by running the notebooks.

## Datasets

The datasets are public datasets and are not included in this repository.

Download the datasets from their official sources:

1. DIBaS Dataset  
   https://github.com/gallardorafael/DIBaS-Dataset

2. Dataset-Bacteria  
   https://github.com/MBJamshidi/Bacteria-2033Images-33Types-dataset?tab=readme-ov-file

Only selected foodborne-pathogen-related species are used. The selected species are grouped into binary morphology labels:

- Cocci: `Enterococcus faecalis`, `Enterococcus faecium`, `Staphylococcus aureus`
- Bacilli: `Clostridium perfringens`, `Escherichia coli`, `Listeria monocytogenes`, `Pseudomonas aeruginosa`

After downloading the datasets, place them according to the path settings in the configuration cells near the top of the notebooks. If your local folder names are different, update only the path variables in those configuration cells.

## Setup

Create and activate a Python environment first. For example, using Conda:

```bash
conda create -n fyp2 python=3.10
conda activate fyp2
```

Install the main Python packages:

```bash
pip install -r requirements.txt
```

If PyTorch installation fails, install the CUDA build using the command recommended by the official PyTorch installation page for CUDA 12.8, then rerun the remaining package installation.

Before running GAN notebooks, confirm that the Windows machine can access:

- `python`
- `git`
- `cl`
- CUDA through PyTorch

This is checked in:

```text
notebooks/01_setup_gpu.ipynb
```

## Method notes

- Source images are split before patches are extracted.
- Species names are used only for stratified splitting, auditing, and checking species-level behaviour inside the binary task.
- The CNN output remains binary.
- GAN-generated cocci images are used only to balance the training data.
- Test sets contain real held-out patches only.
- Original microscope noise is kept; the images are not denoised or cleaned.

## Reproducibility notes

The repository includes final manifests and small result summaries for checking. Full reruns require the public datasets and a CUDA-capable GPU.

Checkpoint files are not included because they are too large for normal GitHub storage. To reproduce the full experiment, run the notebooks in order and allow the notebooks to regenerate the required intermediate files.
